# clusters/

Flux CD entrypoint — one subdirectory per cluster.

This layer contains only Flux `Kustomization` resources: they tell Flux *what* to sync and *from where*, but contain no application manifests themselves. Think of it as the wiring between Flux and the rest of the repo.

```
clusters/
  raspi/              ← single cluster (Raspberry Pi 5, server node)
    flux-system/      ← auto-generated by flux bootstrap, do not edit manually
    apps.yaml         ← Kustomization: syncs ./apps
    infrastructure.yaml  ← Kustomization: syncs ./infrastructure/traefik
    cluster-vars.yaml    ← Kustomization: syncs ./infrastructure/cluster-vars
```

## Why separate from `infrastructure/`?

`clusters/` answers "what to deploy and where" (Flux control plane).
`infrastructure/` contains the actual manifests that get deployed.

This separation makes it straightforward to add a second cluster: create `clusters/<new-cluster>/` pointing at the same `apps/` and `infrastructure/` directories, with cluster-specific overrides only where needed.
